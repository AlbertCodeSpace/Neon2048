# Neon2048 · 霓虹 2048

经典小游戏 **2048** 的霓虹风复刻版。基于 **Cocos Creator 3.8.8**（TypeScript）开发，面向竖屏移动端，目标分辨率 **750×1334**。

滑动合并相同数字的方块，目标是拼出 **2048**——每个数值对应一种霓虹色，合并、获胜时还有粒子爆闪与音效。

---

## 玩法

- **滑动 / 方向键**：上、下、左、右移动所有方块。
- 两个**数值相同**的方块相撞会**合并**为它们的和，并得分。
- 每完成一次有效移动，会在空格随机生成一个新方块（`2` 或 `4`）。
- 拼出 **2048** 即获胜（可继续挑战更高分）；棋盘填满且无法再移动时游戏结束。

## 特性

- 🎨 **霓虹配色**：`2 → 2048` 每个数值一种荧光色，越大字号自适应缩小避免溢出。
- ✨ **动效与庆祝**：`tween` 驱动滑动动画，合并 / 2048 达成时触发代码生成的星光粒子爆发（`StarBurst`）。
- ↩️ **撤销**：最多回退 **5** 步（`UNDO_LIMIT`）。
- 💾 **本地存档**：分数、最高分与当前棋盘通过 `localStorage` 持久化，刷新后自动续局。
- 🔊 **音效与 BGM**：循环背景音乐 + 点击 / 滑动 / 合并 / 获胜 / 撤销等一次性音效，支持一键静音（静音状态也会记住）。
- 🖱️ **多端输入**：同时支持键盘方向键与触摸 / 鼠标滑动（滑动阈值 `30px`）。

## 操作

| 输入 | 动作 |
| --- | --- |
| ← ↑ → ↓ 方向键 | 移动方块 |
| 滑动（触摸 / 鼠标拖拽） | 移动方块 |
| 撤销按钮 | 回退一步（上限 5 步） |
| 保存按钮 | 手动保存当前进度 |
| 重开按钮 | 重新开始一局 |
| 喇叭按钮 | 静音 / 取消静音 |

## 运行与构建

本项目**没有** `npm` / 命令行构建脚本——一切都在 **Cocos Creator 编辑器**中进行（编辑器读取 `package.json` 的 `creator.version` 来匹配引擎版本）。

1. 用 **Cocos Creator 3.8.8** 打开本项目目录。
2. 打开场景 `assets/scene/Game.scene`，点击 **Preview** 预览。
3. 构建发布：**Project → Build**，已提供 `web-mobile` 两套 Web 构建配置，输出到 `build/`。

> **自定义插屏**：Cocos 的自定义插屏依赖在线授权校验，构建时偶发回退为默认 Logo。本项目通过构建插件 `extensions/splash-fix/` 在构建后自动改写输出中的 `splashScreen` 强制使用自定义插屏；也可在构建后手动运行 `node tools/patch-splash.js` 作为兜底。

## 项目结构

游戏逻辑全部位于 `assets/scripts/`，遵循「脚本在运行时按节点名查找并 `addComponent`」的约定，编辑器手动连线极少。

| 文件 | 职责 |
| --- | --- |
| `GameLogic.ts` | 纯 2048 规则，**零引擎依赖**，便于单元测试；`move(dir)` 返回每个方块的运动轨迹、合并与新生成。 |
| `GameManager.ts` | 控制器（挂在 `Canvas`）：解析场景节点、实例化方块、驱动逻辑、动画、输入、存档与撤销。 |
| `GameConfig.ts` | 共享常量与霓虹色板（`GRID`、`MOVE_TIME`、`UNDO_LIMIT`、`localStorage` 键等）。 |
| `Block.ts` | 方块视图，按数值重新着色背景 / 辉光并选择字号。 |
| `PopupCtrl.ts` | 弹窗控制（游戏结束 / 保存 / 重开三种变体）。 |
| `AudioManager.ts` | 全局音频单例，运行时从 `resources/audio` 加载所有音效，持久化静音状态；业务侧用 `sfx('name')` 触发。 |
| `MuteButton.ts` | 切换静音并交换喇叭图标。 |
| `StarBurst.ts` | 代码驱动的星光粒子爆发，用于合并 / 获胜庆祝。 |

## 技术栈

- **引擎**：Cocos Creator 3.8.8
- **语言**：TypeScript（`strict: false`，装饰器 API）
- **平台**：Web 移动端（竖屏 750×1334）
- **持久化**：`localStorage`
